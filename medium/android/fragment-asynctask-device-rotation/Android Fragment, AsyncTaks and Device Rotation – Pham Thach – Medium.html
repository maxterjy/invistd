Handle Fragment and AysncTask on Device Rotation in Android

Pham Thach
Jun 5

Problem
We are building a download file application.
When DownloadFragment is created, it create a DownloadAsyncTask to perform download operation in background.

Issue 
When device rotates, fragment is destroyed and created again, that also make a new DownloadAsyncTask instance.
It is wasteful because the first instance already do the job.

Solution

Step 1: Set retainInstance = true to retain DownloadFragment across the rotation, so fragment onCreate called only once.

Step 2: Get the retained DownloadFragment instance from fragment stack
